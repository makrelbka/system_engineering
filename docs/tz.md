### 1 Цель
- Создать систему онлайн-заказа цветов: каталог, корзина, оформление заказов и админ-панель на базе Vue + Spring Boot + PostgreSQL.
- Сделать простой интерфейс, чтобы клиент без аккаунта мог быстро собрать заказ, а админ видел все заказы/заявки.

Общая идея: нужен понятный и аккуратный онлайн-сервис для заказа цветов, где всё максимально просто — человек заходит, сразу видит красивые карточки букетов, быстро добавляет нужные позиции в корзину и оформляет заказ без регистрации и лишних шагов. Хочется, чтобы интерфейс выглядел аккуратно и “по-человечески”: понятные подписи, крупные кнопки, минимум полей, всё читается с телефона. Клиенту важно быстро понять цену, увидеть фото, легко исправить количество и получить подтверждение, что заказ принят. Администратору нужна отдельная панель, где он сразу видит все заказы и заявки, может оперативно менять статусы и управлять ассортиментом (добавлять/удалять товары и фото) без технических сложностей. В целом это должен быть небольшой, удобный сервис “зайти — выбрать — заказать”, с фокусом на простоту и скорость.

### 2 Внешний вид и работа
1. Главная страница: карточки товаров (название, описание, цена, изображение).
2. Корзина: изменение количества/удаление, отображение суммы.
3. Форма оформления: имя, телефон, email, адрес и кнопка “Оформить заказ”.
4. Небольшая форма callback (имя + телефон).
5. Админка: вход по email/паролю, табы “Заказы” и “Заявки”.

### 3 Техническая модель
1. Фронтенд: Vue 3 + Vite + Tailwind, подгружает каталог через `/api/products`.
2. Бэкенд: Spring Boot, REST API, PostgreSQL, JPA.
3. Каталог товаров хранится в базе данных.
4. Админские аккаунты и заявки на звонок хранятся в базе данных.
5. Метрики: Spring Actuator + Prometheus, Grafana отображает графики.
6. Docker Compose для backend/frontend/postgres/prometheus/grafana.

### 4 Технический функционал
1. Работа с каталогом:
   - Показать покупателю весь список товаров.
   - Дать детальную карточку товара с описанием, ценой и фото.
2. Заказы:
   - Позволить оформить заказ, собрав данные клиента, адрес и выбранные позиции.
   - Давать админу общий список всех заказов для обработки.
3. Callback:
   - Принимать короткие заявки на обратный звонок (имя и телефон).
4. Админка:
   - Логин по email/паролю.
   - Табы заказов: новые, в работе, доставляются, завершённые; возможность менять статус.
   - Табы заявок: новые и завершённые; возможность отмечать обработанные.
   - Управление пользователями админки: добавить/удалить.
   - Управление товарами: добавить новый товар с фото, удалить товар.

### 5 Сущности
- Product: название, описание, цена, изображение (URL или PNG).
- Order и OrderItem со статусами: новый → в работе → доставляется → завершён.
- CallbackRequest: имя, телефон, статус.
- User (админ): email, пароль.
