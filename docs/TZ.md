# Техническое задание
## Система онлайн-заказа цветов "FlowerShop"

### 1. Общая информация

**Название системы:** Система онлайн-заказа цветов "FlowerShop"

**Цель создания:** Автоматизация процесса приема и обработки заказов на доставку цветов. Система предназначена для упрощения взаимодействия между клиентами и цветочным магазином, позволяя клиентам самостоятельно просматривать каталог товаров, формировать заказы и отслеживать их статус в режиме реального времени.

**Краткое описание:** Веб-приложение для приема и обработки заказов на доставку цветов. Клиенты имеют возможность регистрироваться в системе, просматривать каталог цветов и композиций, добавлять товары в корзину и оформлять заказы с указанием адреса доставки. Администраторы системы управляют каталогом товаров, обрабатывают заказы и отслеживают статистику продаж.

### 2. Функциональные требования

**Роли пользователей:**
- **Клиент** — пользователь, который просматривает каталог и оформляет заказы
- **Администратор** — пользователь, который управляет товарами и обрабатывает заказы

**Функции для клиента:**
- Регистрация и авторизация в системе
- Просмотр каталога цветов с возможностью фильтрации по категориям
- Просмотр детальной информации о товаре (название, описание, цена, изображение)
- Добавление товаров в корзину с указанием количества
- Оформление заказа с указанием адреса доставки и контактных данных
- Просмотр истории своих заказов и их текущего статуса
- Редактирование корзины (изменение количества, удаление товаров)

**Функции для администратора:**
- Управление каталогом товаров (создание, редактирование, удаление)
- Управление категориями товаров
- Просмотр всех заказов в системе
- Изменение статуса заказа (новый, в обработке, доставляется, выполнен, отменен)
- Просмотр статистики продаж по категориям и товарам

### 3. Нефункциональные требования

**Производительность:**
- Время отклика API не должно превышать 500 мс для 95% запросов
- Система должна поддерживать до 1000 одновременных пользователей

**Безопасность:**
- Аутентификация пользователей через JWT токены
- Пароли должны храниться в зашифрованном виде (bcrypt)
- Разделение доступа по ролям (клиент/администратор)

**Надежность:**
- Система должна быть доступна 99% времени
- Автоматическое логирование ошибок и критических событий

**Масштабируемость:**
- Архитектура должна позволять горизонтальное масштабирование
- Использование микросервисной архитектуры для независимого масштабирования компонентов

### 4. Основные сущности системы

- **Пользователь (User)** — информация о клиентах и администраторах системы (email, пароль, имя, телефон, роль)
- **Категория (Category)** — категории товаров (букеты, комнатные растения, горшечные цветы и т.д.)
- **Товар (Product)** — цветы и композиции с описанием, ценой и изображением
- **Заказ (Order)** — заказ клиента со статусом, общей стоимостью и адресом доставки
- **Элемент заказа (OrderItem)** — товар в заказе с указанием количества и цены на момент заказа
- **Корзина (Cart)** — временное хранилище выбранных товаров до оформления заказа
